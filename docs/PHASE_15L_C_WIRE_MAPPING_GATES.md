# Phase 15L-C — Wire Mapping Protocol Gates into Promotion/Ingest

**Status: safety wiring only. NOTHING ingested, NO candidate approved, NO mapping
fabricated, NO calibration, NO forecast change. Official ledger unchanged
(6 training / 0 holdout / 0 pending). Phase 15E remains BLOCKED.**

Phase 15L-B built the Observed Outcome Mapping Protocol as an *isolated, advisory*
validator. Phase 15L-C **wires it into the Phase 15J promotion/ingest path** so the
factory cannot create an official OBSERVED case unless a reviewer-authored,
gate-passing mapping justifies the four-bucket distribution.

The official ledger schema/loader/ingest, the 15L-B protocol module, the 8

## Gate behavior (per target)

| situation | result |
|---|---|
| **no mapping**, target training/holdout | **BLOCKED** — "target requires a reviewer-authored, gate-passing ProposedOutcomeMapping" |
| **`direct_observed_distribution`** (gates pass) | observed built from the mapping (real denominator + `high`/cited confidence); `provenance=measured_four_bucket`; **counts toward the ≥20 bar** |
| **`assumption_labeled_distribution`** (gates pass) | ingests as training, **forced `low` confidence**, `provenance=assumption_based_labeled`; **0.0 toward the direct bar** (capped to a minority) |
| **`action_anchor_only`** / **`evidence_only`** | **BLOCKED** — "cannot produce an official observed distribution" |
| **`reject`** | **BLOCKED** — cannot promote to any observed target |
| **any** target=holdout | **BLOCKED** — a retrospective known outcome can never be a clean holdout (G7) |
| **pending** | no mapping required; a distribution mapping on pending is **BLOCKED** |

Additional hard gates (all enforced before any write): mapping `candidate_id` must
match the candidate (no id-laundering); the mapping must be reviewer-authored
(`reviewer` + `reviewed_at` set — no autogenerated mappings); a failing
`validate_mapping` (G1/G2/G4/G5/G6/G8/G11…) blocks with **no `--force` escape**; a
candidate's own `claimed_outcome_proportions`, if present, must agree with the
mapping. The candidate's `ReviewerChecklist` must already be complete (15J).

## The new human promotion workflow

```bash
cd apps/api && export PYTHONPATH=src
# 1. inspect the candidate + its maximal-honest mapping type
python scripts/phase_15j_candidate_factory.py show --id <ID>
python scripts/phase_15l_mapping_protocol.py classify --candidate-id <ID>
# 2. complete the candidate's reviewer_checklist (edit validation_cases/candidates/<ID>.json)
python scripts/phase_15j_candidate_factory.py validate --id <ID>
# 3. draft a mapping proposal
python scripts/phase_15l_mapping_protocol.py mapping-template --candidate-id <ID> \
    --out validation_cases/mapping_proposals/<ID>.json --dry-run   # preview, then write
# 4. fill it in, then validate against the hard gates
python scripts/phase_15l_mapping_protocol.py validate-mapping --from validation_cases/mapping_proposals/<ID>.json
# 5. approve + ingest (the mapping is auto-resolved by candidate id; dry-run first)
python scripts/phase_15j_candidate_factory.py approve --id <ID> --target training
python scripts/phase_15j_candidate_factory.py ingest  --id <ID> --dry-run
python scripts/phase_15j_candidate_factory.py ingest  --id <ID>
```

A proposal that passes `validate-mapping` is still only a proposal; **approval is
the factory's job** (the completed checklist + the explicit `approve`), never a
field on the proposal (`human_approved` is pinned `False`).

## Why the current 8 candidates remain blocked

All 8 classify as `action_anchor_only` (6) or `evidence_only` (2) — a buyer
numerator over a self-selected denominator. None can supply a
`direct_observed_distribution` (no representative/census denominator), so none can
produce an official observed distribution today. A human *could* author an
`assumption_labeled_distribution` (explicit priors) for some, but that ingests
only as a down-weighted, capped, `low`-confidence case and **never** counts toward
the direct-observed bar. No mapping has been authored; nothing is ingested.

## Why Phase 15E is still blocked

Readiness is now mapping-quality-aware: the ≥20 bar counts **only**
`direct_observed_distribution` cases (`ledger_direct_observed_count` reads the
persisted provenance). Current value: **0 direct-observed cases**, **0 clean
holdout** (prospective-only), **0 Tier-1/2 outcome cases**, plus the
`weak_mapping_warning` (Kickstarter concentration, Pebble non-independence). Phase
15E stays **BLOCKED** until ≥20 direct-observed cases + ≥1 clean holdout + ≥1
Tier-1/2 outcome, with assumption-labeled held to a minority.
